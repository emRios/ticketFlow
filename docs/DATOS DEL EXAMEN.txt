Requerimientos funcionales

Módulo de usuarios con roles: Administrador, Agente, Cliente.

Creación, edición y seguimiento de tickets.

Asignación automática de tickets considerando la carga actual de cada agente.

Etiquetado de tickets para categorización rápida.

Tablero visual tipo Pipedrive con drag & drop entre estados: Nuevo, En Proceso, En Espera, Resuelto.

Registro de bitácoras y trazabilidad completa de acciones.

Notificaciones asíncronas ante cambios de estado.

Requerimientos técnicos

Implementar colas/mensajería (RabbitMQ, Kafka, SQS o equivalente).

Transacciones asíncronas para tareas en segundo plano.

Logs y bitácoras con trazabilidad completa (quién, cuándo, qué cambió).

Código modular, estructurado y documentado.

Pruebas unitarias en componentes clave.

Publicar el código en un repositorio GitHub (público o privado).

Entregables

README con instalación, configuración y ejecución.
